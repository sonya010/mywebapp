## 2. Документація по розробленому веб-застосунку
**Призначення застосунку:** mywebapp — це REST API сервіс для відстеження задач (Task Tracker). Дозволяє переглядати список задач, створювати нові та відмічати існуючі як виконані.

**Як налаштувати середовище для розробки:**
1. Встановити Node.js та PostgreSQL.
2. Встановити залежності: `npm install`.
3. Налаштувати параметри підключення до БД у файлі `config.json`.
4. Виконати міграцію бази даних: `node migrate.js`.

**Як запустити веб-застосунок локально:**
Виконати команду `node app.js`. Застосунок буде доступний за адресою `http://localhost:8000`.

## 3. Документація по API-ендпоінтах
* `GET /` - Повертає HTML-сторінку зі списком усіх ендпоінтів.
* `GET /tasks` - Виводить усі задачі. Підтримує text/html та application/json залежно від заголовку Accept.
* `POST /tasks` - Створює нову задачу. Тіло запиту має містити поле `title`.
* `POST /tasks/<id>/done` - Змінює статус задачі на "done".
* `GET /health/alive` - Перевірка стану (повертає HTTP 200 OK).
* `GET /health/ready` - Перевірка підключення до БД (повертає HTTP 200 OK).

## 5. Запуск за допомогою Docker Compose (Лабораторна робота №2)

Проєкт було повністю контейнеризовано. Для автоматичного запуску всіх трьох сервісів (PostgreSQL, Node.js App, Nginx) використовується Docker Compose.

**Інструкція з запуску:**
1. Переконайтеся, що на вашому комп'ютері встановлено Docker та Docker Compose.
2. Склонуйте репозиторій та перейдіть у папку проєкту.
3. Виконайте команду для запуску сервісів у фоновому режимі:
   ```bash
   docker-compose up -d
   ```
