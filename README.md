## 2. Документація по розробленому веб-застосунку
**Призначення застосунку:** mywebapp — це простий REST API сервіс для відстеження задач (Task Tracker). Дозволяє переглядати список задач, створювати нові та відмічати існуючі як виконані.

**Як налаштувати середовище для запуску:**
1. Встановити Node.js.
2. Встановити залежності командою: `npm install`
3. Запустити базу даних PostgreSQL (наприклад, через Docker на порту 5433 для локальної розробки).
4. Налаштувати параметри підключення до БД у файлі `config.json`.
5. Виконати міграцію бази даних (створення таблиць): `node migrate.js`

**Як запустити веб-застосунок:**
Виконати команду: `node app.js`
Застосунок буде доступний за адресою `http://localhost:8000`

## 3. Документація по API-ендпоінтах
* `GET /` - Кореневий ендпоінт. Повертає HTML-сторінку зі списком усіх ендпоінтів.
* `GET /tasks` - Виводить усі задачі (id, title, status, created_at). Підтримує формати HTML та JSON.
* `POST /tasks` - Створює нову задачу. Тіло запиту має містити поле `title`.
* `POST /tasks/<id>/done` - Змінює статус задачі з вказаним `<id>` на "done".
* `GET /health/alive` - Перевірка стану застосунку (повертає HTTP 200 OK).
* `GET /health/ready` - Перевірка підключення до бази даних (повертає HTTP 200 OK, якщо БД доступна).
